### 根本原因
`run_simulation.cu`期望的API与现有物理模块不一致：

| 模块 | 问题 |
|------|------|
| **LaserSource** | 期望RAII类，实际是POD结构 |
| **ThermalLBM** | 缺少`collision()`便捷方法 |
| **FluidLBM** | `applyDarcyDamping()`签名不匹配 |
| **VTKWriter** | 期望Builder模式，实际是静态工具类 |

## 四、风险管理

### 高风险点

#### 风险1：LaserHeatSource与ThermalLBM耦合
**症状**: `applyHeatSource()`需要访问内部分布函数
**缓解**: 两步调用
```cpp
laser->updateHeatSource();
thermal.addHeatSource(laser->getHeatSource(), dt);
```

#### 风险2：VTKWriter缺少PhaseState字段
**症状**: `thermal->getPhaseState()`可能不存在
**缓解**: 临时使用静态方法或从液相分数计算

### 失败回退策略
- 阶段0-2失败：检查现有代码完整性
- 阶段3-4失败：降级使用现有API（手动管理内存）
- 阶段5-6失败：检查API调用，逐个修复
